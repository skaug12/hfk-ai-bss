AI를 쓸수록, 나는 '빈머리'가 되는 걸까?

최근 한 멤버와 운영자가 나눈 대화입니다. AI를 쓸수록 내 실력은 어떻게 되는지에 대한 이야기였습니다.

대기업 20년차 마케터 멤버

요즘 에이전틱으로 24시간 일한다는 게, 정말 '나'의 능력이 좋아진 걸까요? 그냥 '에이전트'가 좋아진 거 아닐까. 나는 오히려 빈머리가 되는 거 아닐까 싶어요.

HFK

스킬을 고도화하는 건 내 업무를 아주 구체적으로 발견하고 다듬는 일이에요. 인풋을 잘 넣어야 아웃풋인 에이전트가 잘 나오니까, 인풋 넣는 일을 게을리할 수가 없어요. 오히려 머리가 단단해지는 기분이에요.

좋은 문장을 쓰려고 안 보던 소설까지 봐요. 그걸 에이전트로 만들려면 결국 글로 써내야 하고, 그 과정에서 지식이 내 것이 되더라고요. 클로드 코드가 저를 멱살 잡고 성장시키는 중이에요. 결국 암묵지가 차별화를 만듭니다.

좋은 인풋, 곧 내 암묵지를 넣어야 좋은 아웃풋이 나옵니다. 스킬을 다듬는 과정이 내 일을 다시 들여다보는 공부가 됩니다.

Why Claude Code

Claude vs Claude Code

같은 Claude라도 대화창에서 쓰는 것과 코드를 직접 실행하는 것은 결과가 다릅니다.

Claude (대화형 AI)

  • 생각하고 답을 만드는 엔진
  • 파일 업로드 → 요약/분석 결과를 텍스트로 제공
  • 데이터가 크면 샘플링하거나 잘림 발생
  • 중간 과정을 검증하기 어려움 (블랙박스)
  • 결과를 다시 수작업으로 옮겨야 함

Claude Code (코딩 에이전트)

  • Claude라는 엔진에 파일을 읽고 코드를 실행하는 손발을 달아준 도구
  • 전체 데이터를 빠짐없이 처리 (샘플링 없음)
  • 코드가 남아 검증/재현 가능 (투명한 과정)
  • 결과를 파일 저장, API로 바로 반영
  • 스킬로 저장해 다음에 한 번에 재실행

정확히는: 같은 Claude 모델을 '채팅 인터페이스'로 쓰느냐, '코딩 에이전트(Claude Code)'로 쓰느냐의 차이입니다.

한계 이해

AI는 왜 틀릴까: 같은 뿌리, 다른 증상

둘 다 '다음에 올 말을 확률로 예측해' 글을 씁니다. 그래서 그럴듯하지만 틀린 답(할루시네이션)이 나옵니다. 뿌리는 같고, 틀렸을 때 드러나는 방식이 다릅니다.

Claude (채팅)의 실수

  • 무엇을 어떤 과정에서 틀렸는지 알 수 없다
  • 숫자와 사실을 그럴듯하게 지어낸다
  • 검증할 수단이 없어 교정 기회가 적다
  • 위험: 그럴듯한 오답을 사실로 믿게 된다

Claude Code (실행)의 실수

  • 틀리면 에러와 실패 기록이 남는다
  • 코딩 시행착오를 반복하며 코드를 고친다
  • 위험: 통과는 하는데 틀린 미묘한 버그가 있을 수 있다

한두 번 물어가며 정답에 가까워지는 건 시간 끌기가 아니라, 실행해보고 정보를 모아 추측을 교정하는 과정입니다. 검증 수단(테스트·에러·실제 파일)을 줄수록 빨라집니다.

비유

같은 데이터 분석을 맡기면: 두 사람 비유

같은 매출 표에 '지난달 대비 증감률 뽑아줘'라고 시켜봅니다. 둘은 서로 다른 종류의 실수를 합니다.

Claude (채팅) = 암산 천재, 검산은 안 함

  • 큰 표를 끝까지 안 보고 머릿속으로 어림한다
  • '약 12% 증가한 것 같아요' (실제로는 9%)
  • 틀려도 자신 있게 말해 티가 안 난다

실수 유형: 맞는 질문에 틀린 답

Claude Code = 검산은 완벽, 가끔 엉뚱한 걸 계산

  • 코드로 정확히 9.3% 산출 (다시 돌려도 같은 값)
  • 단, '지난달'에 취소 주문을 넣었다면?
  • 그 잘못된 전제로 정확한 9.3%를 낸다

실수 유형: 틀린 질문에 맞는 답

그래서 사람이 챙길 것이 다릅니다: 채팅에는 검산(숫자가 맞는지)을, Claude Code에는 전제 확인(무엇을 계산할지)을 짚어줘야 합니다.

하네스

Workshop 1 / 4

웹 크롤링으로 기록 모으기 API·MCP 없이

목표: 흩어진 글과 자료를 Claude Code가 웹에서 가져와 한 폴더에 정리합니다.

준비물: 모으고 싶은 사이트 주소, 저장할 폴더 이름. 따라 입력만 하면 됩니다.

STEP 1

목록부터 만들기

> 나는 경쟁사 블로그를 매주 손으로 확인해. https://example.com/blog 글 제목·링크·날짜를 표로 모아줘.

STEP 2

본문 저장하기

> 각 글 본문을 마크다운으로 ~/Documents/크롤링에 저장해줘. 1개만 먼저 시험 저장해 보여줘.

STEP 3

깔끔하게 정리

> 이 작업을 /collect-blog 스킬로 만들어줘. 다음엔 주소만 주면 같은 방식으로 모으게.

완성: 크롤링 폴더에 글마다 마크다운 파일과 목록 표가 생깁니다. 공개된 페이지만 가져오고, 사이트의 robots 규칙을 지키세요.

스킬 기반: Claude Code의 웹 가져오기 기능과 Playwright(브라우저 자동화)로 페이지를 열어, '본문만 추려서 마크다운으로 저장해줘' 프롬프트를 더해 만듭니다.

Workshop 2 / 4

유튜브로 브리핑 노트 만들기 API 연결

목표: 재생목록에 영상을 넣으면, 그 영상 내용을 요약한 브리핑 노트가 자동으로 만들어집니다.

준비물: 유튜브 재생목록 하나, /analyze-youtube 스킬

STEP 1

영상 담기

브리핑 받고 싶은 영상을 'AI브리핑' 재생목록에 추가합니다.

STEP 2

브리핑 요청

> 출퇴근에 본 강의가 기억이 안 나. 'AI브리핑' 재생목록 새 영상을 요점 5줄·인상 문장 3개·내 업무 적용 2개로 정리해줘.

STEP 3

노트로 저장

> 결과를 ~/Documents/유튜브노트에 저장하고, 이 과정을 /brief-youtube 스킬로 만들어줘.

완성: 영상마다 요점, 인용, 적용 아이디어가 담긴 브리핑 노트가 쌓입니다. 자막이 없는 영상은 음성에서 자동으로 받아씁니다.

스킬 기반: 유튜브 자막을 가져오는 기능에 '요점만 정리해줘' 프롬프트를 묶은 /analyze-youtube 스킬입니다.

Workshop 3 / 4

기록을 액션 플랜으로, 캘린더까지 MCP 연결

목표: 쌓인 메모를 정리해 할 일 목록을 만들고, 그대로 캘린더에 일정으로 넣습니다.

준비물: 정리할 노트 폴더, Google Calendar 연결(MCP)

STEP 1

할 일만 뽑기

> 회의 끝나면 할 일을 자꾸 놓쳐. ~/Documents/회의록 노트에서 내 할 일만 '누가·무엇을·언제까지'로 뽑아줘.

STEP 2

살 붙이기

> 각 할 일에 예상 소요시간·마감일을 붙이고 급한 순서로 정렬해줘. 마감 없으면 '미정'.

STEP 3

캘린더 등록

> 마감일은 구글 캘린더에 넣는 것까지 /meeting-todo 스킬로 만들어줘. 등록 전 항상 미리 보여주게.

완성: 액션 플랜 문서가 생기고, 마감일 일정이 캘린더에 올라갑니다. 먼저 미리보기만 요청해 확인한 뒤 등록하세요.

스킬 기반: 노트를 읽는 파일 기능과 Google Calendar 연결(MCP)에 '할 일만 뽑아 일정으로 넣어줘' 프롬프트를 더해 만듭니다.

Workshop 4 / 4

인스타그램 카드뉴스 만들어 올리기 API 연결

목표: 글 하나로 인스타그램 카드뉴스를 만들고 예약 발행까지 합니다.

준비물: 소재가 될 글이나 주제, /post-to-buffer 스킬(Buffer 연결)

STEP 1

장면 쪼개기

> 인스타 카드뉴스 만들기가 오래 걸려. 이 글을 6장으로 나눠줘. 1장 후킹, 2~5장 한 메시지, 6장 행동 유도.

STEP 2

이미지로

> 각 장을 1080x1080 이미지로 ~/Documents/카드뉴스에 저장해줘. 첫 장 시안 먼저 보여줘.

STEP 3

예약 발행

> 글만 주면 예약 발행까지 하는 /make-cardnews 스킬로 만들어줘. 발행 전 미리보기.

완성: 카드 이미지 세트와 예약된 인스타 포스팅이 만들어집니다. 첫 장 후킹 문구가 가장 중요합니다. 좌상단부터 읽히게 두세요.

스킬 기반: Playwright로 카드 HTML을 1080x1080 이미지로 렌더링하고, Buffer 예약 발행 API로 올리는 /post-to-buffer 스킬입니다.

Hands-on

직접 해보기: 하루를 자동 기록하는 /run-daily-note 스킬 만들기 API·MCP 없이

미션: 오늘 한 작업을, 어떤 프롬프트로 무슨 결과가 나왔는지까지 데일리 노트 한 장에 그대로 남기는 스킬을 만드세요.

준비물: Claude Code가 도는 작업 폴더. 프롬프트는 '상황 구체화 → 결과물 형식 → 스킬화' 순서로. 폼이 아니라 대화예요: 한 칸 친 뒤 Claude 답을 보고 다음 칸으로 (앞 내용은 기억하니 다시 안 써도 됨).

STEP 1

작업 내역 자동 수집

> 오늘 한 일을 자동으로 모아줘. 이 대화에서 한 작업, 오늘 만지거나 만든 파일들, git을 쓰면 오늘 커밋까지 전부 훑어서 정리해줘.

STEP 2

프롬프트·답변·결정·결과 남기기

> daily/오늘날짜.md 에 오늘 작업마다 ①내가 보낸 프롬프트 원문 ②Claude 답변·처리 ③의사결정과 이유 ④결과를 그대로 남겨줘. 산출물은 표로. 1장 먼저 보여줘.

STEP 3

스킬로 만들기

> 이 수집-정리 과정을 /run-daily-note 스킬로 만들어줘. 하루 끝에 부르면 그날 작업을 같은 형식의 데일리 노트로 자동 기록하게.

완성: 부를 때마다 그날 작업이 ① 내 프롬프트 → ② Claude 답변·처리 → ③ 의사결정 → ④ 결과 순으로 그대로 남습니다. 나중에 펼치면 그날 작업이 재현됩니다.

Gen AI 보안 리스크 관리

AI를 업무에 활용할 때 반드시 알아야 할 보안 원칙입니다.

공유해도 되는 것

  • 코드, 개발 관련 질문
  • 일반적인 업무 내용, 기획 아이디어
  • 마케팅 카피, 콘텐츠 초안

피해야 하는 것

  • 비밀번호, API 키, 인증 토큰
  • 주민등록번호, 계좌번호, 카드번호
  • 고객 개인정보 (이름+연락처+주소 조합)
  • 미발표 일정, 공개 전 내부 전략

1. 데이터 격리

민감 정보는 .env 파일로 분리, .gitignore로 업로드 차단

2. 대화는 휘발성

대화는 나와 Claude만 볼 수 있고, 끝나면 Claude도 잊어버림

3. 로컬 실행

Claude Code는 내 컴퓨터에서 실행. 파일 내용은 처리를 위해 Anthropic 서버로 전송되지만 저장되지 않음

Cost Tips

토큰, 아껴 쓰는 법

토큰 = Claude가 읽고 쓰는 글자 단위. 매 질문마다 지금까지의 대화 전체를 다시 읽습니다. 대화가 길어질수록 한 번 질문에 쓰는 토큰이 눈덩이처럼 불어납니다.

1. 낡은 세션 정리: /compact

/compact = 지금까지의 대화 내용을 요약본으로 압축하는 명령어.

  • 대화가 길어졌을 때 입력하면 과거 내용을 짧게 줄여줌
  • 주제가 바뀔 때마다 한 번씩 실행하면 효과적
  • 아예 새 대화를 시작하는 것도 방법

한 세션에서 모든 걸 하려 하지 마세요. 주제별로 나누는 게 낫습니다.

2. Claude가 읽을 범위 좁히기

Claude는 '어디를 봐야 하는지' 모르면 폴더 전체를 탐색합니다. 파일 경로와 범위를 명확히 지정하세요.

  • ✗ '이 프로젝트 개선해줘' → 수십 개 파일 탐색
  • ✓ 'src/auth.ts 파일의 login 함수 수정해줘' → 파일 1-2개만 읽음

경로가 명확할수록 토큰이 줄고, 결과도 정확해집니다.

3. 기본 모델 바꾸기

기본값은 Sonnet. 복잡한 작업에만 Opus로 전환해서 사용.

/model sonnet 으로 전환

4. 작업 전: /status

큰 작업을 시작하기 전에 /status로 현재 세션 정보 확인. 대화가 길어졌다면 /compact 또는 새 세션.

5. 작업 중: /cost

/cost로 현재 세션에서 사용한 비용 확인. Pro/Max 구독이면 API 비용 없이 사용 가능.

첫번째 지능: 타고난 능력

  • 학습, 분석, 판단 등 인지적 능력

Hands-on · 가볍게 시작

가볍게 시작: STAR로 내 업무를 뜯어보고, 당장 쓸 스킬 만들기 API·MCP 없이

미션: 자료가 없어도 괜찮다. STAR로 내가 하는 일을 세세하게 파악하고, 그중 하나를 당장 직장에서 쓸 스킬로 만든다.

준비물: 없음. STAR 프레임과 5분이면 됩니다. STEP 1은 Claude가 질문을 하나씩 던지는 왕복 대화예요. 여러 번 주고받은 뒤 다음 칸으로.

STEP 1

STAR로 내 업무 파악

> 내가 직장에서 자주 하는 업무 하나를 STAR로 뜯어보고 싶어. Situation·Task·Action·Result를 한 번에 하나씩 물어봐서, 이 일이 실제 어떤 단계로 굴러가는지 구체적으로 정리하게 도와줘.

STEP 2

당장 쓸 결과물 정하기

> 정리된 STAR에서 매번 반복되는 단계 하나를 골라, 내가 다음에 바로 쓸 결과물(주간 보고 초안·회의록 정리·메일 답안 등)로 만들어줘.

STEP 3

스킬로 만들기

> 이 과정을 /내업무이름 스킬로 만들어줘. 직장에서 그 일이 생길 때마다 부르면 같은 결과물이 나오게.

완성: STAR로 파악한 내 업무가 당장 쓰는 스킬이 됩니다. 남의 사례가 아니라 내 일에서 출발한 첫 스킬.

Core Concept

커스텀 스킬

마크다운(.md) 파일 하나로 Claude에게 복잡한 업무 절차를 가르칠 수 있습니다.

만드는 법

  1. .claude/commands 폴더에 .md 파일 생성
  2. 파일 안에 업무 절차를 자연어로 작성
  3. Claude Code에서 /파일명 으로 실행

잘 만드는 팁

  • 단계별로 명확하게 절차를 기술
  • 입출력 형식과 예시를 포함
  • MCP 도구명을 명시하면 정확도 향상
  • 에이전트 스킬로 여러 스킬을 연결 가능
  • GitHub에 올려 팀과 공유 가능
Action

지금까지 본 것을 바탕으로: 내 워크스페이스 만들기

다양한 케이스를 봤습니다. 앞으로 내 업무에 맞는 환경을 만들어봅시다.

Step 1

반복 업무 목록화

매주/매달 반복하는 업무를 적어보기. 시간이 많이 드는 것부터 우선순위 결정

Step 2

첫 번째 스킬 만들기

.claude/commands 에 .md 파일 하나 생성. 절차를 자연어로 작성

Step 3

필요한 서비스 연결

내가 자주 쓰는 서비스를 MCP로 연결. Claude Code 공식 문서 참고

Step 4

스킬을 조합해 에이전트로

개별 스킬이 쌓이면 여러 스킬을 묶어 에이전트 스킬로 확장

코딩을 몰라도 됩니다. 업무 절차를 자연어로 적으면 Claude가 이해합니다. 반복되는 업무가 있다면, 그것이 바로 첫 번째 스킬 후보입니다.

Philosophy

심플하게, 이것 하나만 제대로

AI 서비스가 넘쳐나는 시대

ChatGPT, Gemini, Copilot, Claude, Cursor, n8n, Make, Zapier... 매주 새로운 도구가 등장합니다.

타인의 템플릿이나 스킬을 받아도, 책을 사는 것만으로 내 지식이 되지 않듯, 실제로 실행해봐야 합니다.

'코드를 실행해보고, 왜 이 코드를 만들었을까? 그럼 나는 어떻게 해볼까? 까지 고민을 전개해보세요.'

HFK Philosophy

AI = 기본기  ,   HFK = 인사이트와 센스

AI만 있으면?

  • 누구나 빠르게 기본기를 갖추게 된다
  • 평균의 아웃풋이 올라간다
  • 차별화가 어려워진다
  • AI = 평범함의 도구가 될 수도 있다

AI + 인사이트/센스가 있으면?

  • 기본기를 AI로 빠르게 처리
  • 남는 시간에 인사이트와 센스를 키운다
  • 업계 지식 + AI 실행력 = 진짜 강점
  • HFK는 그 인사이트, 센스를 함께 키운다

기본기를 AI로 빠르게 마스터하고, 비즈니스 인사이트와 센스, 즉 자신만의 암묵지를 기르는 게 필요합니다.
