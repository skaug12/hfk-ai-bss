왜 T인가? S가 "하던 일을 더 잘"이었다면, T는 "안 해봤던 일에 발을 담그는 것". AI 덕분에 진입 문턱이 크게 낮아진 영역입니다.

T in Detail

T = Try, 평소 못한다고 생각했던 것

"전공이 아니라서", "툴을 몰라서", "배운 적 없어서"로 그동안 손대지 못했던 일을 AI를 옆에 두고 한번 해보는 영역입니다.

T 영역의 특징

  • 결과물이 남습니다: S와 달리 습관이 아니라 산출물 중심
  • 첫 시도는 무조건 엉성합니다: 완벽을 목표로 하면 시작을 못 함
  • "처음 해봅니다"를 AI에게 알려주는 게 관건: 맥락이 바뀜
  • 한 번 뚫으면 가능 영역이 크게 넓어집니다: 전문 분야의 진입 문턱이 내려감

파트너의 T 영역 사례 (실제 시도)

  • /edit-premiere-xml: 영상 편집 약함 → 프리미어 XML을 텍스트로 편집
  • /update-card-news: 디자인 비전공 → 카드뉴스를 반복 개선
  • Apps Script: 프로그래밍 처음 → 구글폼 자동 알림 위젯
  • /ga-report: 데이터 분석 약함 → GA4 월별 리포트 자동화

공통점: 전부 "이건 내 분야가 아니야"에서 시작했습니다.

T in Detail

T 영역 스킬은 어떻게 뚫는가

T 영역 공통 4단계

  1. 맥락: "이 분야 처음입니다. 기본 개념부터 알려주세요."
  2. 작은 목표: "완벽한 결과" 대신 "일단 한 번 돌아가는 것"
  3. 관찰: 첫 결과의 어디가 어색한지 구체적으로 지목
  4. 반복: "이것만 고쳐보자"를 3~5번

/insights를 옆에 두기

3회차에 써본 /insights로 내 프롬프트 습관을 점검하세요. "왜 결과가 엉뚱한가"의 답이 대개 내 질문 방식에 있습니다.

예시: 어설픈 첫 프롬프트에서 출발

# 맥락
데이터 분석은 처음. 표만 겨우 봅니다.
# 작은 목표
엑셀 한 장을 막대그래프 1개로.
# 원하는 결과 형식
HTML 파일 1개, 브라우저에서 바로 실행.
# 체크 방법
코드 전에 뭘 준비해야 하는지 먼저 알려줘.

기억할 것
T 영역의 가장 큰 장벽은 기술이 아니라 "난 못할 것 같다"는 감정입니다. AI에게 "처음입니다"를 말하는 순간 문이 열립니다.

Workshop · Step 1

STEP 1: 못한다고 생각했던 것 떠올리기

"못한다"의 뜻

  • "내 전공이 아니라서" 미뤄둔 것
  • "툴을 배운 적이 없어서" 시도 못 한 것
  • "전문가한테 맡겨야 하는 일" 같아 포기한 것
  • "나는 저런 건 못해"라고 습관적으로 말한 것

업무도 좋고 취미도 좋습니다 (디자인 · 영상 · 코딩 · 데이터 분석).

잘 안 떠오르면

  • 옆 사람과 5분 짝 토론: "요즘 손 안 가는 일" 서로 묻기
  • /insights로 내 AI 사용 패턴에서 약한 영역 찾기
  • 슬라이드의 4가지 영역(아카이빙 · 연동 · 메일 · 루틴)에서 고르기

"뭘 할지 모르겠다"는 회고가 많았습니다. 못 고르는 게 정상입니다.

활동: 노트에 3개를 적어보세요. "못할 것 같은 이유"도 한 줄씩.

Workshop · Step 2

STEP 2: 가장 가벼운 1개 고르기

고르는 기준 3가지

  1. 실패해도 남에게 피해 안 가는 것: 회사 공식 결과물 말고 내 실험물
  2. 결과물이 한 장 / 한 파일로 나오는 것: 범위가 좁을수록 좋음
  3. 검색만으로는 배우기 벅찼던 것: 옆에서 알려줄 사람이 필요했던 영역

회사 데이터가 막혀 있다면

오늘 T는 개인 · 일상 영역으로 고르면 됩니다. 보안 걱정 없이, 내 자료로 도전.

하지 말 것

  • "한 달 걸리는 프로젝트"를 T에 넣지 마세요
  • "전문가 수준으로 마무리"를 목표로 잡지 마세요
  • 여러 명이 같이 봐야 하는 결과물은 오늘 보류

활동: 가장 작은 범위의 일 1개에 도전해봅니다.

Workshop · Step 3

STEP 3: 첫 명령 만들기

T 영역 4칸 양식

  1. 맥락: "이 분야 처음입니다. 기본부터 알려주세요."
  2. 작은 목표: "일단 한 번 돌아가는 결과 1개"
  3. 원하는 결과 형식: "HTML 파일 / 표 1개 / 노트 1장"
  4. 체크 방법: "결과 보여주고 물어봐주세요"

폼이 아니라 대화입니다. 한 칸 친 뒤 답을 보고 다음 칸으로.

완성 예시: "인스타 저장글 아카이빙 처음 해보기"

# 맥락
코딩 처음. 인스타에 저장만 잔뜩 해둠.
# 작은 목표
저장한 글 1개를 텍스트로 뽑아 노트에 저장.
# 원하는 결과 형식
마크다운 노트 1장. 제목 + 본문 + 링크.
# 체크 방법
1개만 먼저 해보고 결과를 보여줘.

활동: 노트에 4칸을 그리고 각각 한 줄씩 채워보세요. "처음입니다"를 꼭 쓸 것.

Workshop · Step 4

STEP 4: 실행 + 3번 피드백

실행 방법

  1. VS Code 클로드 코드 채팅창 열기
  2. 4칸 내용을 일상 언어로 풀어서 입력
  3. 첫 결과를 받음: "엉성할 것을 예상"
  4. 어디가 어색한지 구체적으로 1~2개 피드백
  5. 3번째 반복까지만. 그 이상은 다음에

자주 막히는 곳

  • "잘 모르겠어요" → 선택지 3개를 제시해달라고 요청
  • 결과가 엉뚱함 → 맥락을 더 자세히 (왜 처음인지)
  • 한 번에 완벽하게 하려는 마음 → "다음 1개만 고쳐줘"로 좁히기

입력 예시 (그대로 써도 됨)

> 코딩은 처음이고, 인스타 저장글을
  텍스트로 모아본 적이 없어.
  저장글 링크 1개를 줄 테니
  제목+본문을 마크다운 노트로 만들어줘.
  먼저 1개만 해서 보여줘.

> (결과 받은 뒤)
  본문이 잘렸어. 전체로 다시.
  저장 폴더는 ~/Documents/아카이브로.

3번째 피드백까지만. 그 다음엔 멈추고 기록하세요.
