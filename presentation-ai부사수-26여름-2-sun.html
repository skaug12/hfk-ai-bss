클로드 코드란?

내 말을 이해하는 개발자. 한국어로 말하면 알아서 코드를 설계해 문제를 해결해주는 AI.

ChatGPT는 "알려주는" AI, 클로드 코드는 "실행하는" AI.

스킬과 에이전트

스킬 = 레시피
정해진 순서대로 실행

에이전트 = 요리사
스스로 판단하며 여러 단계를 처리

반자동화 vs 완전자동화

클로드 코드는 반자동화 방식. 중간중간 "이렇게 할까요?" 확인하며 진행.

매니저로서 프로세스를 직접 점검하고 주도.

Obsidian → 클로드 코드(106개 스킬) → 아임웹 쇼핑몰 → Slack·캘린더. 반복되는 운영 업무를 반자동화.

FAQ

자주 묻는 질문

Q. 계정 하나로 컴퓨터 몇 대까지 쓸 수 있나요?

본인이 쓰는 컴퓨터라면 여러 대도 괜찮습니다. "몇 대까지"라는 정해진 숫자는 없습니다. 회사 노트북과 집 데스크톱을 오가며 같은 계정으로 쓰는 건 문제없습니다. 단, 다른 사람과 계정을 나눠 쓰는 건 금지입니다. 그리고 여러 대에서 동시에 많이 돌려도 사용량 한도는 한 계정에서 함께 빠져나갑니다.

이어 쓰기

대화 기록은 컴퓨터마다 따로 저장됩니다

클로드 코드의 대화 기록은 인터넷 계정이 아니라, 지금 쓰는 컴퓨터의 디스크에 파일로 쌓입니다. 그래서 A 컴퓨터에서 나눈 대화가 B 컴퓨터에 저절로 나타나지 않습니다.

무슨 뜻이냐면

  • 회사 컴퓨터에서 한 대화는 회사 컴퓨터에만 남습니다.
  • 집 노트북을 켜면, 같은 계정이어도 그 대화는 안 보입니다.
  • 계정은 클라우드, 대화 기록은 내 컴퓨터. 둘은 따로입니다.

그래서 좋은 점

내 대화는 내 컴퓨터 안에 있는 내 자산입니다. 회사 계정을 반납하더라도, 파일만 미리 빼두면 다른 컴퓨터에서 그대로 열어볼 수 있습니다.

공부법

강의를 듣고 결정하기보다, 구독하며 공부하기

AI 도구는 영상으로 미리 다 배운 뒤 시작하는 게 아닙니다. 직접 써보면서 익히는 게 훨씬 빠릅니다.

강의부터 듣고 결정
  • 영상으로 개념만 익히고, 구독할지 말지 계속 고민합니다.
  • 막상 손이 잘 안 갑니다.
  • 배운 내용이 며칠 지나면 사라집니다.
구독하고 바로 써보기
  • 월 27,000원으로 내 업무를 직접 시켜봅니다.
  • 오늘 배운 걸 오늘 적용합니다.
  • 손에 남는 실력이 쌓입니다.

운전은 영상으로 못 배웁니다. 차에 앉아 핸들을 잡아야 늡니다. 클로드도 똑같습니다.

비유

같은 데이터 분석을 맡기면: 두 사람 비유

같은 매출 표에 '지난달 대비 증감률 뽑아줘'라고 시켜봅니다. 둘은 서로 다른 종류의 실수를 합니다.

Claude (채팅) = 암산 천재, 검산은 안 함

  • 큰 표를 끝까지 안 보고 머릿속으로 어림한다
  • '약 12% 증가한 것 같아요' (실제로는 9%)
  • 틀려도 자신 있게 말해 티가 안 난다

실수 유형: 맞는 질문에 틀린 답

Claude Code = 검산은 완벽, 가끔 엉뚱한 걸 계산

  • 코드로 정확히 9.3% 산출 (다시 돌려도 같은 값)
  • 단, '지난달'에 취소 주문을 넣었다면?
  • 그 잘못된 전제로 정확한 9.3%를 낸다

실수 유형: 틀린 질문에 맞는 답

그래서 사람이 챙길 것이 다릅니다: 채팅에는 검산(숫자가 맞는지)을, Claude Code에는 전제 확인(무엇을 계산할지)을 짚어줘야 합니다.

Workshop 1 / 4

웹 크롤링으로 기록 모으기 API·MCP 없이

목표: 흩어진 글과 자료를 Claude Code가 웹에서 가져와 한 폴더에 정리합니다.

준비물: 모으고 싶은 사이트 주소, 저장할 폴더 이름. 따라 입력만 하면 됩니다.

STEP 1

목록부터 만들기

> 나는 경쟁사 블로그를 매주 손으로 확인해. https://example.com/blog 글 제목·링크·날짜를 표로 모아줘.

STEP 2

본문 저장하기

> 각 글 본문을 마크다운으로 ~/Documents/크롤링에 저장해줘. 1개만 먼저 시험 저장해 보여줘.

STEP 3

깔끔하게 정리

> 이 작업을 /collect-blog 스킬로 만들어줘. 다음엔 주소만 주면 같은 방식으로 모으게.

완성: 크롤링 폴더에 글마다 마크다운 파일과 목록 표가 생깁니다. 공개된 페이지만 가져오고, 사이트의 robots 규칙을 지키세요.

스킬 기반: Claude Code의 웹 가져오기 기능과 Playwright(브라우저 자동화)로 페이지를 열어, '본문만 추려서 마크다운으로 저장해줘' 프롬프트를 더해 만듭니다.

Workshop 2 / 4

유튜브로 브리핑 노트 만들기 API 연결

목표: 재생목록에 영상을 넣으면, 그 영상 내용을 요약한 브리핑 노트가 자동으로 만들어집니다.

준비물: 유튜브 재생목록 하나, /analyze-youtube 스킬

STEP 1

영상 담기

브리핑 받고 싶은 영상을 'AI브리핑' 재생목록에 추가합니다.

STEP 2

브리핑 요청

> 출퇴근에 본 강의가 기억이 안 나. 'AI브리핑' 재생목록 새 영상을 요점 5줄·인상 문장 3개·내 업무 적용 2개로 정리해줘.

STEP 3

노트로 저장

> 결과를 ~/Documents/유튜브노트에 저장하고, 이 과정을 /brief-youtube 스킬로 만들어줘.

완성: 영상마다 요점, 인용, 적용 아이디어가 담긴 브리핑 노트가 쌓입니다. 자막이 없는 영상은 음성에서 자동으로 받아씁니다.

스킬 기반: 유튜브 자막을 가져오는 기능에 '요점만 정리해줘' 프롬프트를 묶은 /analyze-youtube 스킬입니다.

Workshop 3 / 4

기록을 액션 플랜으로, 캘린더까지 MCP 연결

목표: 쌓인 메모를 정리해 할 일 목록을 만들고, 그대로 캘린더에 일정으로 넣습니다.

준비물: 정리할 노트 폴더, Google Calendar 연결(MCP)

STEP 1

할 일만 뽑기

> 회의 끝나면 할 일을 자꾸 놓쳐. ~/Documents/회의록 노트에서 내 할 일만 '누가·무엇을·언제까지'로 뽑아줘.

STEP 2

살 붙이기

> 각 할 일에 예상 소요시간·마감일을 붙이고 급한 순서로 정렬해줘. 마감 없으면 '미정'.

STEP 3

캘린더 등록

> 마감일은 구글 캘린더에 넣는 것까지 /meeting-todo 스킬로 만들어줘. 등록 전 항상 미리 보여주게.

완성: 액션 플랜 문서가 생기고, 마감일 일정이 캘린더에 올라갑니다. 먼저 미리보기만 요청해 확인한 뒤 등록하세요.

스킬 기반: 노트를 읽는 파일 기능과 Google Calendar 연결(MCP)에 '할 일만 뽑아 일정으로 넣어줘' 프롬프트를 더해 만듭니다.

Workshop 4 / 4

인스타그램 카드뉴스 만들어 올리기 API 연결

목표: 글 하나로 인스타그램 카드뉴스를 만들고 예약 발행까지 합니다.

준비물: 소재가 될 글이나 주제, /post-to-buffer 스킬(Buffer 연결)

STEP 1

장면 쪼개기

> 인스타 카드뉴스 만들기가 오래 걸려. 이 글을 6장으로 나눠줘. 1장 후킹, 2~5장 한 메시지, 6장 행동 유도.

STEP 2

이미지로

> 각 장을 1080x1080 이미지로 ~/Documents/카드뉴스에 저장해줘. 첫 장 시안 먼저 보여줘.

STEP 3

예약 발행

> 글만 주면 예약 발행까지 하는 /make-cardnews 스킬로 만들어줘. 발행 전 미리보기.

완성: 카드 이미지 세트와 예약된 인스타 포스팅이 만들어집니다. 첫 장 후킹 문구가 가장 중요합니다. 좌상단부터 읽히게 두세요.

스킬 기반: Playwright로 카드 HTML을 1080x1080 이미지로 렌더링하고, Buffer 예약 발행 API로 올리는 /post-to-buffer 스킬입니다.

토큰, 아껴 쓰는 법

토큰 = Claude가 읽고 쓰는 글자 단위. 매 질문마다 지금까지의 대화 전체를 다시 읽습니다. 대화가 길어질수록 한 번 질문에 쓰는 토큰이 눈덩이처럼 불어납니다.

1. 낡은 세션 정리: /compact

/compact = 지금까지의 대화 내용을 요약본으로 압축하는 명령어.

  • 대화가 길어졌을 때 입력하면 과거 내용을 짧게 줄여줌
  • 주제가 바뀔 때마다 한 번씩 실행하면 효과적
  • 아예 새 대화를 시작하는 것도 방법

한 세션에서 모든 걸 하려 하지 마세요. 주제별로 나누는 게 낫습니다.

2. Claude가 읽을 범위 좁히기

Claude는 '어디를 봐야 하는지' 모르면 폴더 전체를 탐색합니다. 파일 경로와 범위를 명확히 지정하세요.

  • ✗ '이 프로젝트 개선해줘' → 수십 개 파일 탐색
  • ✓ 'src/auth.ts 파일의 login 함수 수정해줘' → 파일 1-2개만 읽음

경로가 명확할수록 토큰이 줄고, 결과도 정확해집니다.

3. 기본 모델 바꾸기

기본값은 Sonnet. 복잡한 작업에만 Opus로 전환해서 사용.

/model sonnet 으로 전환

4. 작업 전: /status

큰 작업을 시작하기 전에 /status로 현재 세션 정보 확인. 대화가 길어졌다면 /compact 또는 새 세션.

5. 작업 중: /cost

/cost로 현재 세션에서 사용한 비용 확인. Pro/Max 구독이면 API 비용 없이 사용 가능.

Hands-on · 가볍게 시작

가볍게 시작: STAR로 내 업무를 뜯어보고, 당장 쓸 스킬 만들기 API·MCP 없이

미션: 자료가 없어도 괜찮다. STAR로 내가 하는 일을 세세하게 파악하고, 그중 하나를 당장 직장에서 쓸 스킬로 만든다.

준비물: 없음. STAR 프레임과 5분이면 됩니다. STEP 1은 Claude가 질문을 하나씩 던지는 왕복 대화예요. 여러 번 주고받은 뒤 다음 칸으로.

STEP 1

STAR로 내 업무 파악

> 내가 직장에서 자주 하는 업무를 STAR로 분류하고 싶어. Start(미뤄둔 일)·Try(못한다 생각한 일)·Amplify(이미 잘하는 일)·Recover(반복 업무)를 한 번에 하나씩 물어봐서, 어디서부터 AI로 시작하면 좋을지 정리하게 도와줘.

STEP 2

당장 쓸 결과물 정하기

> 정리된 STAR에서 먼저 자동화할 업무 하나를 골라, 내가 다음에 바로 쓸 결과물(주간 보고 초안·회의록 정리·메일 답안 등)로 만들어줘.

STEP 3

스킬로 만들기

> 이 과정을 /내업무이름 스킬로 만들어줘. 직장에서 그 일이 생길 때마다 부르면 같은 결과물이 나오게.

Core Concept

커스텀 스킬

마크다운(.md) 파일 하나로 Claude에게 복잡한 업무 절차를 가르칠 수 있습니다.

만드는 법

  1. .claude/commands 폴더에 .md 파일 생성
  2. 파일 안에 업무 절차를 자연어로 작성
  3. Claude Code에서 /파일명 으로 실행

잘 만드는 팁

  • 단계별로 명확하게 절차를 기술
  • 입출력 형식과 예시를 포함
  • MCP 도구명을 명시하면 정확도 향상
  • 에이전트 스킬로 여러 스킬을 연결 가능
  • GitHub에 올려 팀과 공유 가능

HFK Philosophy

AI = 기본기  ,   HFK = 인사이트와 센스

AI만 있으면?

  • 누구나 빠르게 기본기를 갖추게 된다
  • 평균의 아웃풋이 올라간다
  • 차별화가 어려워진다
  • AI = 평범함의 도구가 될 수도 있다

AI + 인사이트/센스가 있으면?

  • 기본기를 AI로 빠르게 처리
  • 남는 시간에 인사이트와 센스를 키운다
  • 업계 지식 + AI 실행력 = 진짜 강점
  • HFK는 그 인사이트, 센스를 함께 키운다

기본기를 AI로 빠르게 마스터하고, 비즈니스 인사이트와 센스, 즉 자신만의 암묵지를 기르는 게 필요합니다.
