A 사분면 = Amplify

이미 잘하고 있는 일을, AI 부사수로 한 단계 더 깊게.

  • 강점·한계·목표·제약을 4칸으로 명령
  • 새 표준을 만들고 그 표준 위에서 한 번 더
  • "AI가 잘하는 영역"이 아니라 "내가 잘하는 영역"부터

A 워크숍 4단계

  1. 이미 잘하는 영역 떠올리기
  2. 한계 한 줄이 가장 또렷한 1개 고르기
  3. 강점·한계·목표·제약 4칸 명령
  4. 실행 + 새 표준 정착

→ 오늘은 새 표준이 된 작업을 자동화·스킬로 옮겨 시간을 회수합니다.

Trend 1 · 5/15~22

클로드가 양방향이 됩니다: 프로그램이 클로드 안으로, 클로드가 회사 안으로

어도비가 클로드 안으로

  • 포토샵·프리미어·일러스트레이터 같은 어도비 9개 프로그램이 클로드 채팅창에 직접 붙음
  • '사진 색감 보정해줘'라고 말하면 클로드가 그 자리에서 포토샵을 켜서 직접 작업함
  • 어도비 앱을 켜고 메뉴를 찾는 손동작이 사라짐. 말 한마디가 그 자리를 대신함

회사 시스템이 클로드 안으로

  • '회사 안에서만 도는 AI 작업방'(베타 공개): 우리 파일·자료가 회사 밖으로 안 나가는 채로 AI가 일함
  • '사내 자료에 AI가 안전하게 닿는 통로'(시험판): 회사 데이터를 방화벽 안에 그대로 둔 채 클로드가 가져다 씀
  • 그 작업방을 제공하는 회사 4곳: 대용량은 Cloudflare, 상태 유지는 Daytona, 무거운 계산은 Modal, 보안 격리는 Vercel

'앱 안의 AI'였던 흐름이 'AI 안의 앱'으로, 동시에 'AI 안의 회사'로 옮겨가는 중

Trend 2 · 5월 앤트로픽 발표회

클로드 만든 회사 매출 80배, 다음에 낼 카드 3장

앞으로 3가지 방향

  • 코딩 안 깨지게: 멤버북 자동화 같은 실제 일에 맡겨도 사고 없이 끝까지
  • 한 번에 더 많은 자료: 책 여러 권 분량 글을 한 번에 다루는 방향
  • 여러 AI가 함께 일하기: 한 명이 하던 일을 여러 AI가 역할 나눠서 동시에 처리

회사 가치 400억 달러, 매출 80배 성장

새 기능 한 줄씩

  • Claude Design — 클로드가 디자인까지 해줌
  • Claude Code 새 디자인 + 반복 작업 자동 모드
  • Cowork(여럿이 같이 쓰는 공간) 안에서 결과물 실시간 공유
  • Microsoft 365 안에서 바로 쓰는 클로드
  • 일상 도구(메일·달력 등) 연결 모음 확대
  • 가장 똑똑한 모델 Opus 4.7 + 앤트로픽이 직접 관리해주는 AI 비서
Trend 3 · 5/4 두 회사 동시 발표

AI 회사들이 직접 고객사 사무실로 들어갑니다

같은 날 나온 두 발표

  • 5월 4일, OpenAI와 앤트로픽이 비슷한 발표를 동시에 함: 15억·100억 달러짜리 AI 서비스를 기업 현장에 직접 심으러 간다
  • OpenAI는 'AI를 회사 현장에 직접 깔아주는 회사'라는 직군 이름까지 새로 만들어 공식 발표
  • 모델만 만드는 회사가 아니라, 컨설팅 회사처럼 고객사에 직접 들어가 깔아주는 조직으로 변신

팔란티어가 먼저 보여준 형태

  • 팔란티어는 '현장 파견 엔지니어'라는 직군을 2010년대부터 운영. 엔지니어가 고객사 사무실에 직접 가서 그 회사에 맞는 소프트웨어를 그 자리에서 만들어줌
  • 이 직군의 시장 가치가 5년 만에 1,700% 이상 올라감
  • 경쟁 축이 '좋은 AI 모델 만들기'에서 '그 모델을 더 빠르고 깊게 현장에 심기'로 이동

금광 시대에 돈을 번 건 금을 캔 사람이 아니라 곡괭이와 삽을 판 사람이라는 옛 이야기가 다시 나오는 자리

Trend 4 · 1인 창업가 사례

gstack: 1명이 만든 13명짜리 개발팀

누가 어떻게 만들었나

  • 유명 스타트업 액셀러레이터 'Y컴비네이터' 출신 CEO가 60일 동안 혼자 회사 하나 만들 분량을 처리한 비결로 공개
  • 클로드 코드 위에 13명짜리 개발팀을 통째로 올린 형태
  • 기획·디자인·개발·테스트·배포 5단계가 모두 한 줄짜리 명령어로 묶임 (예: 슬래시 입력하고 단어 한 개)

실제 결과물

  • 디자인 참고 사이트(Dribbble) 한 장 → 포트폴리오 사이트 25분 만에 완성
  • 종목 분석 30페이지 발표 자료 5분 만에 자동 생성
  • 코드 한 줄 안 치고 디자인부터 배포까지. 23개 명령어 묶음 가이드는 노션으로 별도 공유

오늘 워크숍에서 만들 자동화의 한 발 앞 모양. 명령어 묶음 하나가 곧 사람 한 명 분량을 흡수함

Trend 5 · 5/13~16

AI 비서가 우리가 쓰던 업무 프로그램 자리로 들어옵니다

헤르메스: 글로벌 1위 오픈소스 AI 비서

'누스 리서치'라는 곳이 공개. GPT·제미나이·클로드 등 200개 이상 AI 모델을 그대로 갈아끼움. 한 회사에 묶이지 않는 게 특징

소상공인용 클로드

해외 회계·결제·고객 관리 프로그램(QuickBooks·PayPal·HubSpot)과 연결. 청구서·마케팅 문구·일정을 클로드 채팅창 안에서 바로 처리

한컴 hwpx 스킬

공문·사업계획서를 한컴 hwpx 파일 그대로 클로드가 만들어줌. 한국 사무 환경에도 클로드가 들어옴

기업에 실제 깔리는 사례: 영업 자동화(Clay), 금융 분석(Rogo), 디자인 자동화(Amplitude). 모두 앞에서 본 '회사 안 작업방·안전한 통로' 위에서 돌아감

Trend 6 · 클로드 사용법

클로드에게 시키는 방식이 다시 바뀝니다

예전 방법은 잘 안 먹힘

  • '넌 10년 경력 전문가야' 같은 역할 부여 명령은 가장 똑똑한 모델 Opus 4.7에선 답변 품질이 거의 안 올라감
  • 대신 '실제 전문가가 어떻게 일하는지 조사한 뒤 그 방식대로 진행해줘'라는 명령에 잘 반응함
  • 예: '실리콘밸리 실무자들의 회의 방식을 깊이 조사하고 학습한 다음 실제 회의처럼 진행해줘'

기본 사용 순서가 바뀜

  • 클로드 안의 '프로젝트' 칸부터 만들어, 관련 자료를 미리 올려두고 시작
  • '맞춤 지침' 칸에 톤·결과 형식을 미리 적어둠 (예: '처음 듣는 얘기처럼 받아써라')
  • 코드 리뷰(CodeRabbit), 개발 환경(JetBrains), 영상 편집(Resolve AI), 터미널(Warp) 같은 도구 안에도 클로드 최신 사용법이 함께 깔리는 흐름

매번 같은 톤 가이드를 다시 설명하던 일을, 이번엔 도구 쪽이 알아서 흡수하는 방향

사례 1 · 5/10

CRM 휴면 멤버 자동 메일링

손이 가던 자리

  • 오랫동안 활동이 없던 멤버에게 "다시 한 번 보러 오세요" 메일을 보내고 싶다
  • 1인 1맞춤 본문이 필요 — 카피만 200명분 못 씀
  • 클릭률을 알아야 다음 톤을 고를 수 있는데 추적 시스템도 없음

R로 회수한 방식

  • T1~T4 시퀀스 템플릿 + HFK 브랜드 컬러, 다크모드 대응
  • HBR Korea Gmail SMTP로 발송 (`send_crm_mail.py`)
  • UTM(`utm_source=crm&utm_campaign=26여름_휴면`)으로 클릭 추적
  • GA4 클릭률 자동 조회 (`crm_check_clicks.py`)
  • 일 50~80명 페이싱, 시범 11명 발송 완료

스킬: /run-crm-mailing · 다음 발송 미리알림: 2026-05-19 09:00

사례 2 · 5/8

아임웹 주문 → 통합 멤버십 시트 5분 sync

손이 가던 자리

  • 아임웹에서 들어온 신청을 시트에 옮기는 일을 매일 손으로
  • STANDBY 상태(결제 대기)와 DELIVERY_WAIT을 따로 봐야 함
  • 옮기다 보면 같은 주문 중복 append, 잊으면 누락

R로 회수한 방식

  • cron_sync.py: 5분마다 fetch · order_no 기준 dedupe
  • SEASON_CATEGORY_CODES 필터로 26여름 주문만
  • launchd com.hfk.imweb-order-sync 자동 실행
  • HFK사람들 시트와 자동 백필(`backfill_hfk_people.py`)

결과: 손이 가던 매일 30분이 0분으로. 누락·중복 사고도 멈춤.

사례 3 · 5/10

어드벤처 댓글 자동 재실행

왜 다시 만들었나

  • 5/10 감각적기획 5회차 어드벤처 댓글 누락 사고
  • 봇이 한 번만 돌고 끝나면, 후속 신청자에게 자동 확정 못 함
  • 운영자가 다시 손으로 트리거해야 했음

R로 회수한 방식

  • state 파일 도입: 마지막 어떤 상태였는지 기억
  • skip 게이트 4종: 과거 세션·메타 공지 시각·빈 신청·diff 없음
  • launchd com.hfk.adventure-auto-rerun 30분 자동
  • 봇이 후속 대기자 자동 확정 (정원 미달 시)

패턴: 사고 한 번 → 메모리만 두지 않고 코드까지

사례 4 · 5/10~13

4L 캔버스 회차 헤더 자동화

손이 가던 자리

  • 매 회차 시작할 때 4L 캔버스 상단에 <N회차. 주제> 헤더 추가
  • 26봄 13개 팀 × 12회차 = 156번 손이 가야 함
  • 저스트두잇 5/12 누락 사고 → 멤버들이 어느 회차인지 못 찾음

R로 회수한 방식

  • 트리거 자동: 봇이 회차 공지 발사 시 캔버스에 즉시 헤더 삽입
  • backfill: 누락된 과거 회차를 시즌 전체에 소급 적용
  • 이중 안전망: 매일 06:00 launchd로 누락분 보충

156번의 손작업 → 트리거 1번 + 안전망 1번 + 소급 1번

사례 5 · 5/13~15

멤버북 10권 InDesign 자동화

손이 가던 자리

  • HFK 에세이 10권(첫 직장~첫 실패), 9명 작가, B6 판형
  • 원본 JD101_최종.pdf (122p) — 손으로 다시 InDesign에 옮기면 며칠
  • 맞춤법 1,500+개를 눈으로 보기엔 너무 많음
  • 인쇄 직전 검수 — 원본 PDF와 1:1 대조 필요

R로 회수한 방식

  • extract.py: IDML → content.json 자동 추출
  • idml_builder.py + make_books.jsx: JSON → 10권 IDML 자동 빌드
  • 맞춤법 xlsx 시스템: x 체크 → 자동 적용 → 처리된 행 취소선 표시 (이번 회차 실시간 사례)
  • diff_pdf_vs_content.py: 원본 PDF ↔ content.json 문장 단위 자동 diff (90글 / 3,406문장)
  • extract_indd_to_json.jsx: InDesign 직접 편집 결과를 다시 SSOT로 (오늘 제작)

이번 회차 라이브: 1,262건 교정 자동 적용 · 245문장 누락 의심 자동 추출

사례 6 · 5/13

"메모리만 두면 또 깨진다" — 코드 자체를 고치기

사고

  • 저자북토크 공지 댓글에 양도 방법 안내 누락
  • 이벤트 footer가 chat_update 시 일부 wipe됨
  • 비슷한 사고가 두 번째

두 가지 선택지

  • (A) 메모리에 "다음엔 footer 확인하라"라고 적기
  • (B) 코드에서 footer 보존 로직 자체를 수정
  • → (B). 메모리는 사람 손을 거쳐야 작동, 코드는 다음에 자동으로 작동

R의 깊은 형태: 손이 가던 일을 줄이는 것뿐 아니라, 사고가 재발할 자리를 코드로 막는 것까지.

사례 8 · 5/14

힉스필드 배치 자동 생성

손이 가던 자리

멤버북 10권 표지 시리즈를 만드는데 한 장씩 프롬프트 넣고 결과 보고 다시 고치고, 톤이 흔들리면 처음부터 다시. 옵션 A·B 두 컨셉을 10권 × 두세 후보씩 굽다 보면 손이 끝없이 가는 일.

R로 회수한 방식

  • JD101 표지 시리즈에선 자동 평가와 베스트 선정, reference 등록까지 묶어서 시리즈 톤을 잠그는 흐름으로 확장

스킬: /run-higgsfield-batch · 기본 대상은 JD101_표지작업/04_표지_컨셉_힉스필드.md

오늘 라이브 · 5/15

방금 만든 도구 둘

맞춤법 결정 xlsx 시스템

  • 1,546개 교정 후보 → CSV로 봤더니 표시한 행이 시각적으로 안 보임
  • CSV → xlsx로 전환, 처리된 행에 취소선 자동 적용
  • --clean 옵션으로 취소선 행 일괄 삭제
  • 다음 검수 때 어디까지 했는지 한눈에

INDD → JSON 추출기

  • InDesign에서 직접 편집한 내용을 다시 SSOT(content.json)로 회수
  • JSX는 READ-ONLY 보장: save/saveAs 미사용, SaveOptions.NO close
  • Python으로 차이 리포트 자동 생성 → 검토 후 수동 교체

두 도구 모두 "같은 손동작을 다음에 또 하지 않기" 위한 것. R의 작은 모양.

Part 3 · R 워크숍

오늘의 R,
반복 하나를 회수합니다

한 명당 한 가지 반복 업무를 골라, 자동화 워크플로우 한 줄로 설계합니다.

Workshop · Intro

오늘 워크숍 흐름

4단계로 진행

  • STEP 1 · 매주 두 번 이상 손이 가는 일 떠올리기
  • STEP 2 · 그중 1개 고르기 (가장 짜증나는 것)
  • STEP 3 · 입력·트리거·출력·실패조건 4칸 명령
  • STEP 4 · 실행 + 안전망 설계

시간 배분 (40분)

  • STEP 1~2: 5분 (혼자)
  • STEP 3: 15분 (Claude와 함께)
  • STEP 4: 10분 (실행)
  • 공유: 10분 (각자 1분)
Workshop · Step 1

STEP 1: 매주 두 번 이상 손이 가는 일 떠올리기

힌트 질문 3개

  • 이번 주에 같은 화면을 두 번 이상 열었던 게 뭐였더라
  • 같은 메시지·메일·문장을 카피해서 붙여본 적은
  • 똑같은 사고가 두 번째인 자리

예시 (이슬기 사례)

  • 아임웹 주문을 시트로 옮기기 · 매일 30분 → 5분 sync
  • 4L 캔버스에 회차 헤더 달기 · 156번 → 트리거 1번
  • 맞춤법 1,500개를 시각적으로 추적 · 무한 → 취소선 1번

3분만 적으세요. 완벽하지 않아도 됩니다.

Workshop · Step 2

STEP 2: 그중 1개 고르기 (가장 짜증나는 것)

3가지 기준

  • 빈도: 주 2회 이상 손이 가는가
  • 짜증: 할 때마다 한숨 나오는가
  • 규칙성: 입력·출력이 매번 비슷한가

고를 때 주의

  • 매번 다른 판단이 필요한 일은 자동화 어려움
  • 1회용 일은 자동화하지 마세요 (그건 R 아닙니다)
  • 사고가 반복된 자리를 우선 후보로
Workshop · Step 3

STEP 3: 입력 · 트리거 · 출력 · 실패조건 4칸 명령

# 4칸 템플릿 (Claude에게 그대로 붙여넣기) 반복 업무: [어떤 일인가, 한 줄로] 입력: [어디서 오는 데이터인가 — 시트/메일/슬랙/폴더] 트리거: [언제 실행해야 하나 — 정해진 시각/이벤트/수동] 출력: [어디로 가야 하나 — 시트/메일/슬랙/파일] 실패조건: [어떤 상황에서 멈춰야 하나 — 빈 데이터/중복/권한] 제약: [절대 하면 안 되는 것 — 예: 기존 데이터 덮어쓰기 금지] → Python 스크립트로 만들어주세요. launchd 등록 명령도 함께.
Workshop · Step 4

STEP 4: 실행 + 안전망 설계

실행 체크리스트

  • 먼저 dry-run 모드로 돌려 보기 (출력 안 보냄)
  • 샘플 1~5건만 실제 실행 → 결과 확인
  • 전체 실행 → 일주일 관찰

안전망 3가지

  • 멱등성: 두 번 돌려도 같은 결과 (중복 방지)
  • 백업: 덮어쓰기 전에 .bak 파일
  • 알림: 실패하면 Slack/메일로 통보

자동화 사고 = 사람 사고보다 빠르고 큼. 안전망이 R의 절반.
