사례 1 · 5/10

CRM 휴면 멤버 자동 메일링

손이 가던 자리

  • 오랫동안 활동이 없던 멤버에게 "다시 한 번 보러 오세요" 메일
  • 1인 1맞춤 본문이 필요 — 카피만 200명분 못 씀
  • 클릭률 추적 시스템도 없음

R로 회수한 방식

  • T1~T4 시퀀스 템플릿 + HFK 브랜드 컬러, 다크모드 대응
  • HBR Korea Gmail SMTP로 발송 (send_crm_mail.py)
  • UTM(utm_source=crm)으로 클릭 추적
  • GA4 클릭률 자동 조회 (crm_check_clicks.py)
  • 일 50~80명 페이싱, 시범 11명 발송 완료

스킬: /run-crm-mailing · 다음 발송 미리알림: 2026-05-19 09:00

사례 2 · 5/8

아임웹 주문 → 통합 멤버십 시트 5분 sync

손이 가던 자리

  • 아임웹 신청을 시트에 옮기는 일을 매일 손으로
  • STANDBY(결제 대기)와 DELIVERY_WAIT을 따로 봐야 함
  • 옮기다 보면 중복 append, 잊으면 누락

R로 회수한 방식

  • cron_sync.py: 5분마다 fetch · order_no 기준 dedupe
  • SEASON_CATEGORY_CODES 필터로 26여름 주문만
  • launchd com.hfk.imweb-order-sync 자동 실행
  • HFK사람들 시트와 자동 백필(backfill_hfk_people.py)

결과: 매일 30분 → 0분. 누락·중복 사고도 멈춤.

사례 3 · 5/10

어드벤처 댓글 자동 재실행

왜 다시 만들었나

  • 5/10 감각적기획 5회차 어드벤처 댓글 누락 사고
  • 봇이 한 번만 돌고 끝나면, 후속 신청자에게 자동 확정 못 함
  • 운영자가 다시 손으로 트리거해야 했음

R로 회수한 방식

  • state 파일 도입: 마지막 어떤 상태였는지 기억
  • skip 게이트 4종: 과거 세션·메타 공지 시각·빈 신청·diff 없음
  • launchd com.hfk.adventure-auto-rerun 30분 자동
  • 봇이 후속 대기자 자동 확정 (정원 미달 시)

패턴: 사고 한 번 → 메모리만 두지 않고 코드까지

사례 4 · 5/10~13

4L 캔버스 회차 헤더 자동화

손이 가던 자리

  • 매 회차 시작할 때 4L 캔버스 상단에 <N회차. 주제> 헤더 추가
  • 26봄 13개 팀 × 12회차 = 156번 손이 가야 함
  • 저스트두잇 5/12 누락 사고 → 멤버들이 어느 회차인지 못 찾음

R로 회수한 방식

  • 트리거 자동: 봇이 회차 공지 발사 시 캔버스에 즉시 헤더
  • backfill: 누락된 과거 회차를 시즌 전체에 소급 적용
  • 이중 안전망: 매일 06:00 launchd로 누락분 보충

156번의 손작업 → 트리거 1번 + 안전망 1번 + 소급 1번

사례 5 · 5/13~15

멤버북 10권 InDesign 자동화

손이 가던 자리

  • HFK 에세이 10권(첫 직장~첫 실패), 9명 작가, B6 판형
  • 원본 JD101_최종.pdf (122p) — 손으로 옮기면 며칠
  • 맞춤법 1,500+개를 눈으로 보기엔 너무 많음
  • 인쇄 직전 검수 — 원본 PDF와 1:1 대조 필요

R로 회수한 방식

  • extract.py: IDML → content.json 자동 추출
  • idml_builder.py + make_books.jsx: JSON → 10권 IDML 자동 빌드
  • 맞춤법 xlsx 시스템: x 체크 → 자동 적용 → 처리된 행 취소선
  • diff_pdf_vs_content.py: 원본 PDF ↔ content.json 자동 diff (90글 / 3,406문장)
  • extract_indd_to_json.jsx: InDesign 직접 편집 결과를 다시 SSOT로

이번 회차 라이브: 1,262건 교정 자동 적용 · 245문장 누락 의심 자동 추출

사례 6 · 5/13

"메모리만 두면 또 깨진다" — 코드 자체를 고치기

사고

  • 저자북토크 공지 댓글에 양도 방법 안내 누락
  • 이벤트 footer가 chat_update 시 일부 wipe됨
  • 비슷한 사고가 두 번째

두 가지 선택지

  • (A) 메모리에 "다음엔 footer 확인하라"라고 적기
  • (B) 코드에서 footer 보존 로직 자체를 수정
  • → (B). 메모리는 사람 손을 거쳐야 작동, 코드는 다음에 자동으로 작동

R의 깊은 형태: 손이 가던 일을 줄이는 것뿐 아니라, 사고가 재발할 자리를 코드로 막는 것까지.

사례 7 · 5/16~17

AI 티 잡아내는 톤 안전망

손이 가던 자리

  • 콘텐츠 만들 때마다 AI 티 표현을 눈으로 다시 확인
  • 같은 톤 가이드를 Claude에게 매번 새로 설명

단어는 한국어인데 화자가 없는 글이 자꾸 새어 나왔습니다.

R로 회수한 방식

  • voice-profile.md를 단일 진입점으로. 화자 좌표, 옵시디언 리뷰 7편을 톤 앵커로, BEFORE/AFTER 10쌍을 통합
  • ai_not_to_do.py 0~100점 자동 검사. 금지어·정형 도입부·양비론 패턴
  • /generate-team-review 스킬에 자가 점검 강제: 70점 미만이면 자동 재작성
  • 이 슬라이드를 검사했더니 처음엔 100점, 좌우 대칭 패턴 결함 발견 후 룰 확장 → 76점

BEFORE · AI 티

이번 세션에서는 리더십에 대해 알아보겠습니다. 본질적으로 자율성이 핵심이라고 할 수 있습니다.

AFTER · 사용자 톤

연초에 세운 운동 계획이 어느새 흐지부지되고, 회사에서 정해진 목표는 왜 해야 하는지 의미가 잘 보이지 않는 경험. 직장인이라면 누구나 한 번쯤 겪어봤을 상황입니다.

실측: 옵시디언 리뷰 7편 평균 94점 (100·100·100·100·95·83·80). AI 티 가상 텍스트는 0점.

사례 8 · 5/14

힉스필드 배치 자동 생성

손이 가던 자리

멤버북 10권 표지 시리즈를 만드는데 한 장씩 프롬프트 넣고 결과 보고 다시 고치고, 톤이 흔들리면 처음부터 다시. 옵션 A·B 두 컨셉을 10권 × 두세 후보씩 굽다 보면 손이 끝없이 가는 일.

R로 회수한 방식

  • 표지 컨셉을 마크다운 한 파일에 코드펜스로 모아두면, 스킬이 그 안의 모든 프롬프트를 추출해 한 번에 4장씩 자동 생성
  • 결과는 옵션별 폴더로 정리되고 인덱스 마크다운이 함께 만들어짐
  • JD101 표지 시리즈에선 자동 평가와 베스트 선정, reference 등록까지 묶어서 시리즈 톤을 잠그는 흐름으로 확장

스킬: /run-higgsfield-batch · 기본 대상은 JD101_표지작업/04_표지_컨셉_힉스필드.md

멤버북 자동화의 jsx를 매번 InDesign 메뉴에서 찾아 여는 게 짜증나서 셸 스크립트 한 줄로 옮겼습니다. 같은 동선이 반복되는 자리.

오늘 라이브 · 5/15

방금 만든 도구 둘

맞춤법 결정 xlsx 시스템

  • 1,546개 교정 후보 → CSV로 봤더니 표시한 행이 시각적으로 안 보임
  • CSV → xlsx로 전환, 처리된 행에 취소선 자동 적용
  • --clean 옵션으로 취소선 행 일괄 삭제
  • 다음 검수 때 어디까지 했는지 한눈에

INDD → JSON 추출기

  • InDesign에서 직접 편집한 내용을 다시 SSOT(content.json)로 회수
  • JSX는 READ-ONLY 보장: save/saveAs 미사용, SaveOptions.NO close
  • Python으로 차이 리포트 자동 생성 → 검토 후 수동 교체

두 도구 모두 "같은 손동작을 다음에 또 하지 않기" 위한 것. R의 작은 모양.

Workshop · Intro

오늘 워크숍 흐름

  • STEP 3: 15분 (Claude와 함께)

Workshop · Step 1

STEP 1: 매주 두 번 이상 손이 가는 일 떠올리기

힌트 질문 3개

  • 이번 주에 같은 화면을 두 번 이상 열었던 게 뭐였더라
  • 같은 메시지·메일·문장을 카피해서 붙여본 적은
  • 똑같은 사고가 두 번째인 자리

예시 (이슬기 사례)

  • 아임웹 주문을 시트로 옮기기 · 매일 30분 → 5분 sync
  • 4L 캔버스에 회차 헤더 달기 · 156번 → 트리거 1번
  • 맞춤법 1,500개를 시각적으로 추적 · 무한 → 취소선 1번

3분만 적으세요. 완벽하지 않아도 됩니다.

Workshop · Step 2

STEP 2: 그중 1개 고르기 (가장 짜증나는 것)

3가지 기준

  • 빈도: 주 2회 이상 손이 가는가
  • 짜증: 할 때마다 한숨 나오는가
  • 규칙성: 입력·출력이 매번 비슷한가

고를 때 주의

  • 매번 다른 판단이 필요한 일은 자동화 어려움
  • 1회용 일은 자동화하지 마세요 (그건 R 아닙니다)
  • 사고가 반복된 자리를 우선 후보로

STEP 4: 실행 + 안전망 설계

실행 체크리스트

  • 먼저 dry-run 모드로 돌려 보기 (출력 안 보냄)
  • 샘플 1~5건만 실제 실행 → 결과 확인
  • 전체 실행 → 일주일 관찰

안전망 3가지

  • 멱등성: 두 번 돌려도 같은 결과 (중복 방지)
  • 백업: 덮어쓰기 전에 .bak 파일
  • 알림: 실패하면 Slack/메일로 통보

자동화 사고 = 사람 사고보다 빠르고 큼. 안전망이 R의 절반.
