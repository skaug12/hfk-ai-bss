Trend 5+ · 깊이 보기

에르메스의 장점과 오픈클로와의 차이

내 질문: '에르메스의 장점은 뭐야? 오픈클로와 차이점은? 내가 에르메스를 쓰면 뭐가 좋아질까? 뭐에 도움될까? 기존 스킬 바탕으로.'

에르메스 장점 4가지

  • 자동 스킬 생성: 작업 해결 시 재사용 가능한 마크다운 스킬 파일을 스스로 작성. 7일 주기 백그라운드 큐레이터가 스킬 라이브러리 정리
  • 영구 메모리: 세션·플랫폼·기기를 넘나드는 지속 메모리. 사용자 프로필을 시간이 지나며 자동 구축
  • 상주형 비서: Mac Mini에 24시간 띄워두고 크론으로 깨워서 작동. 자리에 없어도 진행
  • 완전 로컬: 데이터가 내 머신에만. 텔레메트리·클라우드 종속 없음

오픈클로와의 차이

  • 오픈클로: 프라이버시·폐쇄망 자동화에 강점. 학습/진화는 약함
  • 에르메스: 자기 개선·장기 학습이 강점. 로컬 실행은 가능하지만 폐쇄망 특화는 아님
  • 거칠게 정리: 오픈클로는 '안전하게 시키는 일만 정확히 하는 도구', 에르메스는 '쓸수록 스스로 성장하는 비서'

출처: OpenClo·Hermes·Claude Code 비교 · hermes-agent.org · WikiDocs 실전 가이드 · Felo Search 블로그

Trend 5+ · 내 워크플로우에 맞춰서

기존 HFK 스킬 바탕으로 뭐가 좋아질까

도움될 부분 4가지

  • 반복 작업 자동 스킬화: 시즌별 반복 운영(공지·캘린더·아임웹 sync·도착 체크)에서 스킬화하지 않은 잡일을 매번 손으로 시키는 일이 줄어듦. 패턴 보고 알아서 스킬 생성, 다음번엔 그걸 호출
  • 상주형 운영: 지금 Slack 봇·imweb-order-sync는 launchd로 분리 운영. 에르메스 하나로 통합 + 크론 트리거로 묶을 여지. 다만 어느 컴퓨터에서 도는지 명확히 관리해야 함
  • 메모리 자동 누적: MEMORY.md는 의식적으로 쌓는 구조인데, 에르메스는 자동 누적·정리. 다만 '옵시디언 본문 수정 금지' 같은 부정적 규칙은 자동 학습이 어겨버릴 위험이 있어 이 부분은 수동 관리가 더 안전
  • 로컬 보관: HFK 멤버 정보·Slack 토큰·아임웹 자격증명이 .env에 있는데, 클라우드 텔레메트리 없는 점은 보안상 이점

솔직한 추천

지금 Claude Code 스킬 운영이 잘 굴러가는 상태라 전체 이전은 비추.

대신 반복 트리거형 작업(어드벤처 댓글, 도착 체크, 시즌 데이터 sync) 일부만 에르메스로 옮겨 24시간 상주 테스트해보는 게 현실적 진입점.

코딩·콘텐츠 생성 자체는 Claude Code 강점이 분명하니 그쪽은 유지.

※ 직접 써본 게 아니라 검색 기반 답변임을 전제로.

Trend 6 · 클로드 사용법

클로드에게 시키는 방식이 다시 바뀝니다

예전 방법은 잘 안 먹힘

  • '넌 10년 경력 전문가야' 같은 역할 부여 명령은 가장 똑똑한 모델 Opus 4.7에선 답변 품질이 거의 안 올라감
  • 대신 '실제 전문가가 어떻게 일하는지 조사한 뒤 그 방식대로 진행해줘'라는 명령에 잘 반응함
  • 예: '실리콘밸리 실무자들의 회의 방식을 깊이 조사하고 학습한 다음 실제 회의처럼 진행해줘'

기본 사용 순서가 바뀜

  • 클로드 안의 '프로젝트' 칸부터 만들어, 관련 자료를 미리 올려두고 시작
  • '맞춤 지침' 칸에 톤·결과 형식을 미리 적어둠 (예: '처음 듣는 얘기처럼 받아써라')
  • 암묵지가 점점 중요해짐

매번 같은 톤 가이드를 다시 설명하던 일을, 이번엔 도구 쪽이 알아서 흡수하는 방향

Trend 8 · 메타 대규모 해고

메타 8,000명 해고: 돈이 없어서가 아니라 AI 때문에

차가운 해고 + 역대급 실적

  • 전날 재택 지시, 당일 새벽 4시 이메일로 통보. 일부 직원은 출입 권한 이미 차단된 상태
  • 해고 3주 전 실적: 매출 563억 달러, 순이익 268억 달러(약 37조원)
  • 돈이 없어서 자른 게 아니라는 신호
  • 한 달 전, 직원 노트북에 마우스·키 입력·스크린샷 수집 프로그램 설치('AI 학습용' 명목). 1,000명 이상 반대 청원 후 한 달 뒤 해고 발표

그럼 왜 + 누가 잘렸나

  • AI 인프라에 약 217조원 투자 선언. 돈은 더 크게 쓰지만 그 돈이 기존 직원이 아니라 AI 인프라로 향함
  • 2022~2025 누적 약 33,000명 감원
  • 축소: 혐오 발언 대응, 악성 콘텐츠 검수, 사이버보안, 콘텐츠 무결성
  • 강화: AI 인프라, 파운데이션 모델, AI 연구·개발
  • 조직도가 곧 회사 우선순위

사람이 하던 콘텐츠 감수·플랫폼 안전을 AI가 채우는 흐름. AI 시대에 회사가 무엇을 남기고 무엇을 줄일지 보여주는 장면

사례 1 · 5/10

CRM 휴면 멤버 자동 메일링

손이 가던 자리

  • 오랫동안 활동이 없던 멤버에게 "다시 한 번 보러 오세요" 메일
  • 1인 1맞춤 본문이 필요 — 카피만 200명분 못 씀
  • 클릭률 추적 시스템도 없음

R로 회수한 방식

  • T1~T4 시퀀스 템플릿 + HFK 브랜드 컬러, 다크모드 대응
  • HBR Korea Gmail SMTP로 발송 (send_crm_mail.py)
  • UTM(utm_source=crm)으로 클릭 추적
  • GA4 클릭률 자동 조회 (crm_check_clicks.py)
  • 일 50~80명 페이싱, 시범 11명 발송 완료

스킬: /run-crm-mailing · 다음 발송 미리알림: 2026-05-19 09:00

사례 2 · 5/8

아임웹 주문 → 통합 멤버십 시트 5분 sync

손이 가던 자리

  • 아임웹 신청을 시트에 옮기는 일을 매일 손으로
  • STANDBY(결제 대기)와 DELIVERY_WAIT을 따로 봐야 함
  • 옮기다 보면 중복 append, 잊으면 누락

R로 회수한 방식

  • cron_sync.py: 5분마다 fetch · order_no 기준 dedupe
  • SEASON_CATEGORY_CODES 필터로 26여름 주문만
  • launchd com.hfk.imweb-order-sync 자동 실행
  • HFK사람들 시트와 자동 백필(backfill_hfk_people.py)

결과: 매일 30분 → 0분. 누락·중복 사고도 멈춤.

Workshop · Step 1

STEP 1: 매주 두 번 이상 손이 가는 일 떠올리기

힌트 질문 3개

  • 이번 주에 같은 화면을 두 번 이상 열었던 게 뭐였더라
  • 같은 메시지·메일·문장을 카피해서 붙여본 적은
  • 똑같은 사고가 두 번째인 자리

예시 (이슬기 사례)

  • 아임웹 주문을 시트로 옮기기 · 매일 30분 → 5분 sync
  • 4L 캔버스에 회차 헤더 달기 · 156번 → 트리거 1번
  • 맞춤법 1,500개를 시각적으로 추적 · 무한 → 취소선 1번

3분만 적으세요. 완벽하지 않아도 됩니다.

Workshop · Step 2

STEP 2: 그중 1개 고르기 (가장 짜증나는 것)

3가지 기준

  • 빈도: 주 2회 이상 손이 가는가
  • 짜증: 할 때마다 한숨 나오는가
  • 규칙성: 입력·출력이 매번 비슷한가

고를 때 주의

  • 매번 다른 판단이 필요한 일은 자동화 어려움
  • 1회용 일은 자동화하지 마세요 (그건 R 아닙니다)
  • 사고가 반복된 자리를 우선 후보로
Workshop · Step 3

STEP 3: 입력 · 트리거 · 출력 · 실패조건 4칸 명령

# 4칸 템플릿 (Claude에게 그대로 붙여넣기) 반복 업무: [어떤 일인가, 한 줄로] 입력: [어디서 오는 데이터 — 시트/메일/슬랙/폴더] 트리거: [언제 실행 — 정해진 시각/이벤트/수동] 출력: [어디로 가야 하나 — 시트/메일/슬랙/파일] 실패조건: [어떤 상황에서 멈춰야 하나] 제약: [절대 하면 안 되는 것] → Python 스크립트로 만들어주세요. launchd 등록 명령도 함께.
Workshop · Step 4

STEP 4: 실행 + 안전망 설계

실행 체크리스트

  • 먼저 dry-run 모드로 돌려 보기 (출력 안 보냄)
  • 샘플 1~5건만 실제 실행 → 결과 확인
  • 전체 실행 → 일주일 관찰

안전망 3가지

  • 두 번 돌려도 같은 결과: 중복으로 쌓이지 않게
  • 백업: 덮어쓰기 전에 .bak 파일
  • 알림: 실패하면 Slack/메일로 통보

자동화 사고 = 사람 사고보다 빠르고 큼. 안전망이 R의 절반.
