Part 1 · What's New

최근 몇 주, 무슨 일이 있었나

슬랙에서 @Claude로 일을 맡기는 Claude Tag가 열리고, 클로드 코드가 자기 코드를 스스로 보안 검수하는 플러그인이 나왔습니다.

두 가지 모두 비개발자에게 의미가 있는 변화입니다.

News 1 · Claude Tag

"슬랙에서 @Claude로 일을 맡긴다"

앤트로픽이 자사 제품팀 코드의 65%를 만드는 방식, Claude Tag를 공개. 슬랙에 Claude를 팀원처럼 들여 @Claude로 일을 넘기는 기능. Enterprise·Team 베타, 엔진은 Opus 4.8.

어떻게 동작하나

  1. 말로 일을 던지면 스스로 단계를 쪼개 처리
  2. 결과를 슬랙 스레드에 올림
  3. 채널 맥락을 기억해 팀이 하나의 Claude를 같이 씀
  4. 맡겨두면 며칠짜리 일도 자율로 진행

채널에 물려두는 것

도구·데이터·코드베이스를 채널에 연결해두면 누구나 @Claude를 불러 일을 맡김.

기존 'Claude in Slack'과 다른 점

질의응답에서 그치지 않고, 실제 작업을 끝까지 수행하고 결과를 스레드에 남김. 채팅 상대가 아니라 일하는 팀원에 가까움.

만든 회사가 먼저 의존: 앤트로픽 제품팀 코드의 65%가 이 방식으로 만들어집니다.

왜 이게 의미 있을까

AI를 '도구'가 아니라 '팀원'으로

채널에 들여두고 누구나 일을 맡김. 매니저처럼 위임하는 방식으로 바뀜.

비개발자도 같은 흐름

코드 한 줄 없이, 평소 쓰는 슬랙에서 말로 맡김. 결과는 스레드에 남아 같이 보고 추적.

만든 회사가 먼저 씀

앤트로픽이 자사 제품팀 코드의 65%를 이 방식으로 생산. 시연용이 아니라 실제 업무 도구.

News 2 · Security Guidance

클로드 코드가 자기 코드를 스스로 보안 검수한다

앤트로픽이 클로드 코드용 보안 가이드 플러그인(Security Guidance)을 공개. 코딩하는 동안 위험한 패턴을 백그라운드에서 자동으로 잡아 수정 가이드를 띄움.

무엇을 잡아내나

  • 명령어 주입(command injection), XSS 같은 주요 보안 위험을 자동 감지
  • 위험한 패턴을 백그라운드에서 검사 (코딩 흐름을 끊지 않음)
  • 클로드 코드에서 /plugins 입력하면 플러그인 마켓에서 골라 설치

공식 데모

8초 만에 작성한 145줄 NodeJS 코드 직후 빨간 배너: 'Background security review found'. Go와 Node가 같은 키를 다르게 읽는 틈으로 메시지가 새는 구멍을 발견.

진단에 그치지 않고 허용 키 재구성·변형 키 차단·타입 검사까지 자동 적용.

생각해볼 점
비개발자가 만든 결과물에도 점검은 필요. 사람이 일일이 못 보던 위험을 도구가 대신 잡아주는 방향.

한 가지 더 · /insights

내가 클로드 코드를 어떻게 쓰는지 리포트로 받아본다

채팅창에 /insights 한 줄이면, 최근 30일 동안 내가 클로드 코드를 어떻게 써왔는지 정리한 리포트(HTML)를 만들어줍니다.

무엇을 보여주나

  • 주로 어떤 일에 많이 썼는지 (작업 종류 분포)
  • 세션이 얼마나 효율적이었는지, 토큰은 얼마나 썼는지
  • 자주 막히거나 되돌린 지점 (마찰 포인트)

/usage와 뭐가 다른가

/usage는 지금 이 순간의 비용·사용량을 즉시, /insights는 한 달치 패턴을 분석해 리포트로 묶어줍니다.

왜 좋은가 "내가 클로드를 어디에 쓰고 어디서 막히는지"를 보면 다음에 뭘 스킬로 만들지가 보입니다.

Part 3 — S 사분면
Part 3 · S 사분면 워크숍

오늘은 S부터 시작합니다

STAR 4개 다 할 필요 없습니다. 오늘은 가장 가벼운 한 칸, S = Start만 깊게 다룹니다.

S 사분면이 어디에 있는지 다시 보기

S · Start (오늘 다룸 ★)

하고 싶었는데 미뤄왔던 일 — 습관 형성

T · Try

평소 잘 못한다고 생각했던 것 — 도전

A · Amplify

이미 잘하지만 더 잘하고 싶은 것 — 차별화

R · Recover

반복 작업을 효율화 — 시간 회수

왜 S부터? 제일 부담 없는 칸이기 때문. "잘해야 한다"는 생각 없이 일단 시작하는 것이 핵심.

S = Start — 미뤄왔던 일을 시작하는 칸

"매일 하면 좋은데 자꾸 미루는 것들" — AI에게 그 일을 매일 옆에서 도와달라고 부탁하는 영역.

S 영역의 특징

  • 실패해도 부담이 적음 — 어차피 안 하던 일
  • 성과보다 빈도가 중요 — 매일 짧게가 성공
  • 결과물보다 트리거가 핵심 — "AI가 먼저 말 걸어주는 구조"
  • 나를 위한 일 — 회사 KPI가 아니라 내 삶의 작은 루틴

슬기의 S 영역 4개 (실제 운영 중)

  • /hello-world 아침 캘린더·미리알림·잔여 작업 브리핑
  • /run-daily-note 하루 끝 작업 자동 일지화
  • /cleanup-home 흩어진 다운로드 파일 자동 분류
  • /backup-notes 노트 주기적 백업

공통점: 전부 "매일 해야 하는 줄 아는데 자꾸 까먹는 것"이었음.

S 영역 스킬은 어떻게 작동하는가

예시 1 — /hello-world

아침에 명령 한 줄 입력하면:

  1. 인증 파일 점검
  2. 오늘의 캘린더 일정 가져옴
  3. 미리알림 앱에서 오늘/늦은 항목 가져옴
  4. 어제 작업 일지에서 미완료 항목 확인
  5. 한 화면에 정리해 보여줌

예시 2 — /cleanup-home

홈/다운로드 폴더를 훑고, 파일 종류·날짜로 자동 분류 폴더에 이동.

공통 구조 (4칸)

  1. 트리거 — 언제 시작할지
  2. 데이터 소스 — 어디서 가져올지
  3. 처리 — 어떻게 정리할지
  4. 결과 형식 — 어디로 보여줄지

기억할 것
좋은 S 스킬은 "내가 명령하지 않아도 알아서 말 걸어주는 구조"까지 만드는 것. 오늘은 그 직전 단계인 "명령 한 줄로 시작되는 단계"까지 가봅니다.

준비 — 연결
준비 · 커넥터

클로드 코드에 내 도구를 연결한다: 커넥터

커넥터는 클로드 코드를 구글·슬랙·깃허브 같은 외부 서비스에 연결하는 통로입니다. 한 번 연결해두면 클로드가 그 도구를 직접 불러서 일을 합니다.

무엇을 연결하나

  • 구글(캘린더·드라이브·지메일), 슬랙, 깃허브
  • 직접 만든 도구(MCP 서버)까지

어떻게 추가하나

claude.ai 왼쪽 상단 메뉴 → 왼쪽 탭 [사용자 지정] → 커넥터에서 원하는 서비스를 검색해 설치. 코드 없이 됩니다.

연결해두면 "내 캘린더 일정 가져와", "이 슬랙 채널에 올려줘" 같은 말이 바로 동작합니다.

준비 · 구글 OAuth

커넥터가 편한데도, 구글 키를 따로 받는 이유

왜 필요한가

커넥터는 클로드 코드 안에서 대화할 때만 작동합니다. 매일 자동으로 도는 작업이나 따로 도는 스크립트가 구글에 접근하려면 내 OAuth Client ID & Key가 필요합니다.

무엇을 받나

  • Client ID: ...apps.googleusercontent.com 형태
  • Client Secret: 긴 비밀 문자열 (외부 공유·깃 커밋 금지)

받는 법 (구글 클라우드 콘솔)

  1. console.cloud.google.com 접속 → 프로젝트 만들기
  2. "API 및 서비스"에서 쓸 API 사용 설정 (캘린더·드라이브·지메일)
  3. OAuth 동의 화면 설정 (외부, 앱 이름, 본인 이메일을 테스트 사용자로)
  4. 사용자 인증 정보 → OAuth 클라이언트 ID → 데스크톱 앱 선택
  5. 나온 Client ID·Secret을 JSON으로 저장 → 내 .env에 보관
준비 · 구글 OAuth

받은 키는 대화창에 적지 말고 .env에 넣는다

VS Code에서 바로 .env 열기

클로드에게 ".env에 ID·Secret 넣을 자리를 만들고 그 파일을 열어줘"라고 하면, VS Code 창에 .env가 바로 열립니다. 그 자리에 값을 복사·붙여넣기만 하면 됩니다.

JSON 파일을 받았다면

"다운로드 폴더의 그 JSON 파일을 보고 .env에 기록해줘"라고 시키면 클로드가 알아서 옮겨 적습니다.

딱 하나만 지키기: ID·Secret을 대화창에 직접 적지 않기. "파일을 열어줘 / 보고 기록해줘"라고 시키면 비밀 값이 대화 기록에 안 남습니다.

Workshop · Step 1

STEP 1 — 미뤄둔 일 떠올리기 (5분)

"미뤄둔 일"의 정의

  • "매일 하면 좋은데 안 하고 있는 것"
  • "한 번 시작하면 좋은데 시작이 안 되는 것"
  • "머릿속엔 있는데 손이 안 가는 것"

업무도 좋고, 개인 영역(독서·운동·가계부)도 좋음.

예시 (참고용)

  • 매일 아침 어제 한 일 한 줄로 정리
  • 주말마다 다운로드 폴더 정리
  • 읽고 싶은 기사 매일 한 개씩 요약
  • 매주 월요일 한 주 캘린더 미리 보기
  • 가계부에 어제 결제 내역 기록
  • 슬랙 중요 메시지 모아두기

활동 — 노트를 펴고 3개를 적어보세요. 잘 안 떠오르면 위 예시에서 골라도 됩니다.

Workshop · Step 2

STEP 2 — 가장 가벼운 1개 고르기 (5분)

고르는 기준 3가지

  1. 5분 안에 끝나는 것 — 거창한 건 금지
  2. 결과를 누가 검사 안 하는 것 — 회사 KPI 말고 내 삶의 일
  3. 매일 또는 매주 반복되는 것 — 일회성은 R 영역으로

하지 말 것

  • "내 인생을 바꿀 거대한 습관" 같은 거 고르지 마세요
  • "AI가 알아서 다 해주는" 같은 막연한 기대 금지
  • 다른 사람과 협업이 필요한 것은 보류

활동 — 3개 중 가장 작고 가벼운 것에 동그라미 하나. 그것이 오늘 우리가 만들 S 스킬의 주제입니다.

Workshop · Step 3

STEP 3 — 4칸으로 명령 만들기 (10분)

4칸 양식

  1. 트리거 — "언제 이 일을 할 것인가?"
  2. 소스 — "어디서 정보를 가져올 것인가?"
  3. 처리 — "어떻게 가공할 것인가?"
  4. 결과 — "결과를 어디로 보내줄 것인가?"

한 칸당 한 줄이면 충분. 완벽할 필요 없음.

완성 예시 — "매일 캘린더 정리"

# 트리거
매일 아침 9시
# 소스
구글 캘린더 오늘 일정
# 처리
시간순 정렬, 회의/개인일 분류
# 결과
~/오늘일정.md 파일로 정리

활동 — 노트 한 장에 4칸을 그리고 각각 한 줄씩 채워보세요.

Workshop · Step 4

STEP 4 — 실제로 실행해보기 (15분)

실행 방법

  1. VS Code에서 클로드 코드 채팅창 열기
  2. 4칸 내용을 한국어 문장으로 풀어 입력
  3. 클로드가 묻는 질문에 한 줄씩 대답
  4. 결과 확인 → 잘 되면 다음 칸 추가, 안 되면 같이 풀기

입력 예시 (그대로 써도 됨)

> 매일 아침 9시에
  구글 캘린더 오늘 일정을
  시간순으로 정리하고
  회의/개인일정으로 분류해서
  ~/오늘일정.md 파일로 저장하는
  스킬을 만들어줘.

> 먼저 한 번만 실행해서
  결과를 보여줘.

결과가 마음에 들면 저장해두기. 매일 자동으로 실행하는 건 다음 시간에 다룹니다.

자주 막히는 곳

  • "권한이 없다"는 메시지 → 캘린더/슬랙 등 인증 필요
  • "파일을 찾을 수 없다" → 정확한 폴더 경로 알려주기
  • 너무 많은 걸 한 번에 시키기 → 한 단계씩 나누기

팁: 첫 스킬은 캘린더·슬랙처럼 로그인이 필요한 것보다 내 컴퓨터 파일 정리처럼 바로 되는 일로 시작하면 막힘이 적습니다.
