Introduction

클로드 코드 = 내 말을 이해하는 개발자

아무렇게나 내 마음대로 문제를 입력하면 알아서 코드를 설계해 문제를 해결해주는 AI

클로드 코드는 원래 개발자가 쓰는 도구였음. 더 효율적인 프로그램을 만드는 데 사용했음.

하지만 우리는 다르게 씀. 내 컴퓨터의 파일을 정리하고, 문서를 만들고, 쇼핑몰에 올리는 데 사용함.

'나는 기획자지만, 내 말을 모두 이해하고
문제를 해결해주는 개발자가 생긴 것.'

Why 클로드 코드

ChatGPT, Gemini와의 차이

같은 '200개 상품 데이터 분석해줘'를 요청했을 때의 차이임.

ChatGPT / Gemini : '알려주는' AI

  • 파일 올리면 요약해서 글로 알려줌
  • 데이터가 많으면 일부만 보고 대략 분석
  • 결과를 복사해서 엑셀에 다시 붙여넣어야 함
  • '답변'을 주지만 실행까지는 안 해줌

클로드 코드 : '실행하는' AI

  • 내 컴퓨터의 파일을 직접 열어서 분석
  • 200개 전부 빠짐없이 처리
  • 분석 결과를 쇼핑몰에 바로 반영까지 완료
  • '답변' 대신 '결과물'을 만들어 줌

클로드 코드엔 Claude 포함

클로드 코드에 가입하면 ChatGPT처럼 대화하는 기능이 이미 들어있음.

기존 대화 이전 가능

ChatGPT/Gemini에서 나눈 대화 내용을 옮겨올 수 있음.

구독 하나로 정리

특별히 다른 서비스만의 강점이 없다면, 클로드 코드 하나면 충분함.

# 스스로 판단하며 여러 단계를 처리
Core Concept

요즘 자주 듣는 세 단어: RAG · 하네스 · 아티팩트

RAG · 찾아보고 답하기

답하기 전에 외부 자료에서 관련 내용을 먼저 찾아온 뒤, 그걸 보고 답함. 관련 페이지를 펼쳐 놓고 시험 보는 것과 같음.

나의 경우 = 메모리 + 옵시디언 · 멤버십 시트 · 캘린더

하네스 · 엔진을 자동차로

모델(엔진)을 감싸 실제로 일하게 만드는 바깥 장치. 도구를 쥐여주고, 결과를 돌려받고, 권한을 확인하는 흐름.

나의 경우 = 클로드 코드 + 95개 스킬 + 권한 + MCP

아티팩트 · 남는 결과물

작업이 만들어 내는 실제 산출물. 코드 · 문서 · 이미지 · 발표 자료처럼.

나의 경우 = 레터 · 핸드아웃 · 쇼핑몰 HTML · 공지 · 대시보드

한 줄 그림 · RAG가 무엇을 알고 무엇을 지킬지 공급하고 → 하네스가 받아서 일을 시키고 → 그 결과로 아티팩트가 나옴. 클로드 코드 하나에 이 셋이 다 들어 있음.

Perspective

반자동화 vs 완전 자동화

AI가 알아서 해주는 '자동화'와, 중간중간 내가 확인하는 '반자동화'는 다름.

반자동화 : 클로드 코드 방식

  • Claude가 일하다가 '이렇게 할까요?' 물어봄
  • 내가 확인하고 OK하면 다음 단계로 진행
  • 결과물을 보고 수정 요청도 가능
  • 내가 전체 흐름을 직접 점검할 수 있음

완전 자동화 : MAKE, n8n 등

  • 한 번 설정하면 내가 안 시켜도 알아서 동작
  • 예: 매주 월요일 9시에 리포트 자동 발송
  • 초기 설정이 복잡하고 유지보수에 시간 소요
  • 중간에 뭔가 바뀌면 전체가 깨질 수 있음

파트너의 관점 · 매니저로서 프로세스를 직접 점검하고 주도한다는 측면에서, 반자동화가 좀 더 안전함. 클로드 코드 하나를 야무지게 잘 쓰는 것이, 괜히 자동화까지 건드려 넓고 얕게 쓰는 것보다 낫다는 생각임.

Workflow

수요일 루틴: 한 주 운영을 30분에

'/run-wednesday' 한마디로 아래 4단계가 자동 진행됨. 단계마다 확인을 받고 넘어감.

Step 1

차차주 세션 공지

2주 뒤 세션을 캘린더에서 찾아 팀별 채널에 예약 공지

Step 2

차주 어드벤처 공지

다음 주 어드벤처를 메타 공지에 정원과 함께 안내

Step 3

7일 핸드아웃 생성

수~화 7일치 세션 핸드아웃을 팀별 A4로 일괄 생성

Step 4

to-do-log 정리

남은 할 일을 미리알림과 옵시디언에 주간 단위로 등록

Before · 직접 하면

캘린더 확인 → 공지 작성 → 핸드아웃 디자인 → 할 일 정리

약 4~5시간

After · 클로드 코드로

한마디로 실행 → 중간중간 확인만

약 30분

Toolkit

이걸 떠받치는 도구·라이브러리

대부분 무료이거나 저렴한 표준 도구임. 도구 자체는 평범하고, 이걸 클로드 코드로 엮는 데서 힘이 나옴.

AI 엔진

Claude (Opus · Sonnet). 글쓰기, 분석, 코드 생성을 맡는 두뇌. 클로드 코드가 이 엔진을 부려 씀.

연결 · MCP

Slack · 구글 시트·캘린더 · 아임웹 · 미리알림. 외부 서비스를 직접 제어 (앞 장 참고).

저장 · 호스팅

Cloudflare R2. 상품 상세 이미지를 올려 공개 링크로 쇼핑몰에 바로 사용.

발행 · 디자인

Buffer(SNS 예약 발행) · Figma · 힉스필드(이미지·표지 생성).

자동 실행

launchd · cron. 정해진 시각에 스스로 실행. 봇·시트 동기화가 알아서 돌아감.

만드는 도구

Reveal.js(지금 이 발표) · React + Vite(상품 대시보드) · Python(자동화 스크립트).

기억할 단어 · Claude · MCP · R2 · launchd. 이름은 달라도 하는 일은 분명함. 엔진, 연결, 저장, 자동 실행. 도구를 다 외울 필요는 없고, 필요할 때 클로드 코드에게 물어보면 됨.

Security

AI에게 뭘 말해도 되나요?

클로드 코드는 내 컴퓨터에서 작업하지만, 내가 입력한 내용은 인터넷을 통해 Claude 서버로 전송됨. 그래서 보내는 내용을 구분해야 함.

말해도 되는 것

  • '매출 시트에 컬럼이 5개 있어' (구조만 설명)
  • '이 엑셀을 월별로 분류해줘' (작업 요청)
  • '마케팅 카피 초안 써줘' (콘텐츠 작성)
  • '경쟁사 가격 비교해줘' (공개 정보)

절대 말하면 안 되는 것

  • 고객 개인정보 (이름 + 전화번호 + 주소)
  • 회사 사내 시스템 비밀번호
  • 미공개 재무제표, 인사 정보
  • 주민번호, 카드번호, 계좌번호

비밀번호 보관 방법

Slack 토큰, 쇼핑몰 비밀번호 등은 별도 파일(.env)에 저장함. 클로드 코드가 이 파일을 읽어서 쓰지만, 대화 내용에는 비밀번호가 포함되지 않음.

비밀번호 파일 만드는 순서

  1. 클로드 코드에게 '비밀번호 보관 파일 만들어줘' 요청
  2. 생성된 파일을 직접 열어서 비밀번호 입력
  3. 클로드 코드에게 '비밀번호 파일에서 읽어서 써' 요청

이렇게 하면 대화에 비밀번호가 노출되지 않음.

Philosophy

책을 사는 것만으로 나의 지식이 되지 않듯

직접 해봐야 내 것이 됨

AI부사수 외에도 여러 AI 강의를 들으면, 다른 사람의 템플릿이나 방법을 얻게 됨.

하지만 책을 사는 것만으로 나의 지식이 되지 않듯, 직접 실행해보는 것이 중요함.

다른 사람의 방법을 보고,

  • '왜 이렇게 만들었을까?'
  • '그럼 내 업무에는 어떻게 적용할까?'

까지 생각을 확장해볼 것.

하나를 깊이 있게

ChatGPT, Gemini, Copilot, Claude, Cursor, n8n, Make, Zapier... 매주 새로운 도구가 등장함.

'하나를 정하고, 제대로 깊이 쓰자.'

모으기보다 실행하기

  • 템플릿 10개 모으기보다 1개를 직접 실행
  • 강의 3개 듣기보다 내 업무에 한 번 적용
  • 남의 방법 구경보다 내 방법 하나 만들기
